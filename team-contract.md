## 1) purpose
This contract defines how we work as one team.
Our goal is to finish milestones on time with good quality.

## 2) project scope
We build a dashboard about Mercedes-Benz sales (2020-2025).
Main topics:
- fuel type trends
- model sales ranking
- price and horsepower patterns
- turbo and color patterns

## 3) role split
Each member takes lead on at least one part:
- data and cleaning
- dashboard layout and callbacks
- charts and interaction design
- documents and release checks

All members help with review and testing.

## 4) communication
- we hold at least one team meeting each week
- we post updates in team chat after major progress
- we reply to team messages within 24 hours

## 5) github workflow
- pull latest `main` before starting work
- use branch + pull request for major updates
- request at least one teammate review before merge
- use github issues to track bugs and features
- create milestone release only after team confirmation

## 6) quality rules
- app should run without error
- charts and filters should be clear
- text and labels should be easy to understand
- no sensitive data in repo
- no large raw data file in repo

## 7) responsibility and attendance
If a member cannot finish a task on time, they must tell the team early.
If delay happens:
1. re-assign task
2. update timeline
3. record reason in issue or meeting note

## 8) conflict handling
We discuss conflicts in a meeting with respect.
If needed, we vote as a team.
If still unresolved, we ask the instructor for advice.

## 9) contract updates
Any update to this contract needs agreement from all three members.
